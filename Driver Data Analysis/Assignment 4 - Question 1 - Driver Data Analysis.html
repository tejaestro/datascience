KMeans Clustering¶

Running KMeans algorithm to find the optimal number of clusters using the elbow method

In [8]:
ssq = []

for k in range(1, 11):
    kmeans_model = KMeans(n_clusters = k, random_state = 123)
    kmeans_model.fit(driver_data_scaled)
    ssq.append(kmeans_model.inertia_)

Generating Scree plot to visualise the elbow curve

In [9]:
plt.plot(range(1,11), ssq, marker='o')
plt.xlabel("Number of clusters")
plt.ylabel("Within-cluster SSQ")
plt.title("Scree Plot")
plt.show()

Inference
Looking at the Scree plot we can observe that the curve becomes more or less linear after 4.
Therefore, using this elbow curve we can conclude that 4 would be the optimal number for cluster.

Inference¶

Cluster 1: These are drivers who haven't travelled a lot and haven't oversped as well.
Cluster 4: These are drivers who have had fair amount of travelling but were within the speed limit as far as possible.
Cluster 3: These are drivers who have oversped even though they were travelling short distances.
Cluster 2: These drivers have done the most amount of driving but have also had the most number of overspeeding cases.

We could label these groups as follows -

Cluster 1: Slow pokes
Cluster 4: Balancers
Cluster 3: Sprinters
Cluster 2: Road runners

Going by the thumb rule, we cut the dendrogram where the gap between two successive combination similarities is largest.
So going by the above figure, we can make this cut for 4 clusters.

We verify this numerical choice by calculating the 'Silhouette Score'
